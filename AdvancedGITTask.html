Git Pull Request Methodology Exercises

Topic: Feature Branch Integration Using PR Workflow

Exercise 1: Sequential PR Integration

This exercise demonstrates how two developers integrate their feature branches into the main branch using a pull request methodology. Ana completes her integration first, followed by Luis.

Ana's Workflow (ft_one branch)

  1. GIT - Ana: Did commits #1 and #4 in branch ft_one
  2. GIT - Ana: git checkout main — Switch to the main branch
  3. GIT - Ana: git pull origin main — Fetch and merge the latest changes from remote main
  4. GIT - Ana: git checkout ft_one — Return to the feature branch
  5. GIT - Ana: git rebase main ft_one — Reapply ft_one commits on top of the updated main
  6. GIT - Ana: git add <conflict_file> & git rebase --continue — Resolve each merge conflict and continue the rebase process
  7. GIT - Ana: git push origin ft_one — Push the rebased feature branch to the remote repository
  8. GITHUB - Ana: She creates a new PR with base: main, compare: ft_one and specifies title and description
  9. GITHUB - Reviewer: The reviewer reviews her PR changes and finds some issue with it
  10. GIT - Ana: She does commit #7 to fix this issue
  11. GIT - Ana: git push origin ft_one — Push the fix to update the PR automatically
  12. GITHUB - Reviewer: The reviewer reviews the updated PR and approves it
  13. GITHUB - Ana: She has integrated the changes of ft_one into main correctly

Luis's Workflow (ft_two branch)

  1. GIT - Luis: Did commits #2 and #5 in branch ft_two
  2. GIT - Luis: git checkout main — Switch to the main branch
  3. GIT - Luis: git pull origin main — Fetch and merge the latest changes from remote main (including Ana's merged changes)
  4. GIT - Luis: git checkout ft_two — Return to the feature branch
  5. GIT - Luis: git rebase main ft_two — Reapply ft_two commits on top of the updated main
  6. GIT - Luis: git add <conflict_file> & git rebase --continue — Resolve each merge conflict and continue the rebase process
  7. GIT - Luis: git push origin ft_two — Push the rebased feature branch to the remote repository
  8. GITHUB - Luis: He creates a new PR with base: main, compare: ft_two and specifies title and description
  9. GITHUB - Luis: The reviewer reviews his PR changes and approves them
  10. GITHUB - Luis: He has integrated the changes of ft_two into main correctly
