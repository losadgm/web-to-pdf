Pruebas de Usabilidad

Universidad de Oviedo - Software Y Estandares Para La Web - 2025

Introducción: Condiciones de las Pruebas

La primera prueba (Tanda 1, usuario experto) se realizó en laboratorio de prácticas con XAMPP local y GitHub Pages (limitado). Las Tandas 2-3 se ejecutaron con usuarios remotos accediendo a la versión completamente funcional desplegada en GitHub Pages, permitiendo JavaScript dinámico y APIs sin restricciones. Se probaron 7 usuarios en tableta, teléfono y ordenador, siguiendo protocolo: cronómetro desde "Iniciar Prueba", 10 preguntas sobre navegación/juegos/contenido, observación sin intervención.

Selección de Usuarios

Usuario Experto (Tanda 1): Compañero estudiante ingeniería informática (20 años, hombre).

No Expertos (Tanda 2-3): Profesora primaria (28), socorrista (59), ama de casa (37), promotor ventas (27), estudiante química (21), jubilado (67). Perfiles variados por edad/profesión.

Resultados por Tanda

Tiempo: 71,76s, 7/10 completado, 7/10.

  • GitHub Pages bloquea APIs externas (meteorología) y JS dinámico completo

Solucionado: Nada. No solucionado: Menú colapsable (hamburguesa). Botones "atrás" (útiles para corregir errores sin reiniciar prueba).

Conclusiones Técnicas

El despliegue en un entorno web funcional fue determinante, elevando la tasa de finalización de tareas del 70% (Tanda 1) al 95% (Tandas 2-3) al eliminar las restricciones técnicas iniciales. Se confirmó que los usuarios no expertos priorizan la legibilidad y la usabilidad táctil sobre la complejidad funcional; elementos como el tamaño de fuente resultaron críticos para los usuarios de mayor edad. Las mejoras iterativas aplicadas entre fases validaron el proceso, logrando aumentar la valoración media hasta un 7.7/10 en la última tanda.

Conclusiones Personales

El despliegue real reveló problemas de usabilidad invisibles durante el desarrollo local, demostrando la necesidad de probar en producción. Aprendí a priorizar la corrección de elementos bajo nuestro control (interfaz y navegación) frente a limitaciones externas (datos de APIs) para ser más eficiente. El protocolo de observación no intrusiva con cronómetro fue clave para medir la fricción real, confirmando que el refinamiento incremental por tandas es la estrategia más efectiva para pulir el software.
